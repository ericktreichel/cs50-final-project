   Hello, World! Erick here, and this is my final project for CS50x Introduction to Computer Science course, promoted by Harvard University. I created this web application using mostly Python and SQL to set the back-end, and HTML and CSS (and a bit of Javascript) to build the front-end. As a bonus (or perhaps a compensation for my lack of talent creating the design in general) I've filled some pages with images I have recently created using the AI plataform DALL-E 2: some fun (and maybe weird) portraits of animals that I love (like capybaras and corgis, why not?) having a good time coding.

  The main purpose of this project (besides serving as my final project, of course) is to put into practice much of the knowledge I acquired during all these weeks throughout the course, whether in classes, or in the hundreds of hours spent searching on the internet for solutions to a variety of specific implementations and problems. This site is a mix of a personal blog with a forum, with a programming theme/background. Here users can make a post talking about something they are developing or studying, tell their daily routine or something like that, as they wish.

  • Login control: You can create an account, log in, and then the app will keep track of your navigation through the application, and then you can log out whenever you want.
  • API implementation: APIs are not only helpful, but also inherent in every software or system nowadays, so of course I decided to implement an API in my project; although it's not a very usefull one though. I made use of a free API to show corgi pictures every time you access the Fluffiness Area.
  • Personalization: You are able to add more personal information about yourself than you provided when you created your account, or you can change your current information, including your profile picture. The profile pictures, by the way, once uploaded by the user, will be compressed and only then stored in the database. Users are able to post on the site, edit their posts, and even delete them if they wish.
  • Data persistence: This web app is supported by a database built with SQLAlchemy and Python, which allowed a relatively simple implementation.
  • Different validations: the code will perform multiple data validations when necessary, for instance, checking if the username and e-mail are unique at registration (or when the user tries to change them later, if so), if the passwords match, if all fields were filled and several other validations.
